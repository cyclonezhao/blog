02-智能Agent

发表于 2018-09-03 | 更新于 2018-11-04 | 分类于 AI ， 《人工智能 - 一种现代的方法》笔记

1. Agent与环境


传感器从环境接收感知序列，Agent函数将感知序列转化为行动序列，执行器执行行动序列。

2. 理性

理性涉及以下4个方面：

  1. 性能度量
  2. Agent对环境的先验知识
  3. Agent可以完成的行动
  4. Agent截止到此时的感知序列

3. 任务环境

3.1. 任务环境的规范描述，即PEAS描述：

  1. 性能
  2. 环境
  3. 执行器
  4. 传感器

例如，出租车任务环境的PEAS描述如下：

  1. 性能度量：到达正确的目的地，油量消耗、耗时或费用最小化……
  2. 驾驶环境：各种各样的道路、路上的车辆、行人……
  3. 执行器：油门、刹车、显示或语音输出……
  4. 传感器：摄像头、红外或声呐、GPS……

3.2. 任务环境的性质

  1. 完全可观察的和部分可观察的
  2. 单Agent和多Agent
  3. 确定的和随机的
  4. 片段式的和延续式的：检验次品零件、下棋
  5. 静态的和动态的：填字谜游戏、自动驾驶
  6. 离散的和连续的：国际象棋、自动驾驶
  7. 已知的和未知的：指的是Agent的知识状态

4. Agent程序

4种基本的Agent程序：

  1. 简单反射Agent
  2. 基于模型的反射Agent
  3. 基于目标的Agent
  4. 基于效用的Agent




4.1. 学习Agent


学习Agent划分为4个概念上的组件：

  1. 学习元件：负责改进提高
  2. 性能元件：负责选择外部行动，是前文考虑的整个Agent。
  3. 评判元件
  4. 问题产生器

学习元件的设计很大程度上依赖性能元件的设计，“Agent一旦学会了这个技能，需要何种性能元件行驶该能力”？

评判元件根据固定的性能标准告诉学习元件Agent的运转情况，负责指出Agent的成功程度。应该把性能标准置于Agent之外考虑，理由是Agent不应该修改性能标准来适应自身行为。

问题产生器负责建议探索性行动。如果性能元件自行其是，它会一直根据已知的知识采取最佳的行动。但是，如果Agent可以进行少量探索，做一些短期内可能次优的行动，那么它也许会发现对长期而言更好的行动。
